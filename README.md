# colin-experiment-voice

A markdown skill file for replicating the writing voice of Colin McFarland from his 2012 book *Experiment! Website conversion rate optimization with A/B and multivariate testing*.

## What it is

A structured reference — calibration data, core moves, diction rules, worked examples, and common failure modes — that lets an LLM (or a human editor) write in Colin's register: direct, second-person, conversational, plain English, no hype, no corporate jargon.

## When to use it

- You want crisp, direct prose instead of corporate-flat writing
- You're drafting blog posts, internal memos, or thought leadership
- Someone hands you text full of "leverage," "transformative," and "best-in-class" and asks you to make it human
- You explicitly want the *Experiment!* voice

## How to use it

Feed `colin-experiment-voice.md` as context to your LLM of choice, or use it as a self-editing checklist.

Trigger phrases: "in McFarland's voice", "Experiment! voice", "in the voice of Experiment!"

## What's inside

| Section | What it covers |
|---|---|
| **The voice in one paragraph** | Quick summary of the register |
| **Calibration data** | Sentence length, contraction rate, "you" density, etc. |
| **Core moves** | 12 documented patterns with book examples |
| **Signature phrases** | Openers, transitions, closers |
| **Diction & syntax rules** | Do / Don't lists |
| **What the voice is NOT** | Guard rails against common misreadings |
| **Worked example** | Corporate-flat → Colin's voice, with annotations |
| **Common failure modes** | 7 ways the voice goes wrong |
| **Invocation pattern** | Step-by-step process for writing in voice |
